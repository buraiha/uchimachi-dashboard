# 家まちダッシュボード

家まちダッシュボード は、観光地・地域の「いま何がやってる？」を、さくっと一画面で見られるイベントボードです。  
まずは **イベントスケジュール**だけに機能を絞って、早めに公開して回しながら育てていきます。

- 現地のデジタルサイネージとして掲示
- 一般ユーザーがスマホ等から同じ画面にアクセス

この2つを同じソフトで成立させるのが狙いです。

---

## 目的 / コンセプト

- **イベントが主役**  
  「今日」「直近」「開催中」がひと目で分かる
- **運用がラク**  
  イベント管理は **Google カレンダー**で行い、画面は自動で反映
- **カジュアルで読みやすい**  
  遠目でもスマホでも、サッと把握できるUIにする

---

## 初期スコープ（v0）

### できること
- **リンク付きイベントスケジュール表示**
  - Google カレンダー連動（公開カレンダーを読み込み）
  - タイトル / 日時 / 場所（任意）/ リンク（任意）
  - 「開催中」を強調表示

### やらないこと（後回し）
- 写真カルーセル（ライブカメラ含む）
- 気温表示
- CMカルーセル
- ログイン / 投稿 / コメントなどのユーザー機能
- 自治体サイト等からの自動収集（スクレイピング）

---

## イベント運用ルール（おすすめ）

「リンク付き」を安定させるため、まずは運用ルールでシンプルにします。

- Google カレンダーの各イベントの **説明欄の先頭にURLを1つ置く**
- アプリ側は **説明欄の最初のURL** をイベントリンクとして扱う

※URLが無い場合はリンク無しで表示します。

---

## 想定する画面

- 直近のイベントを時系列で一覧
- 「開催中」バッジで今やってるイベントが分かる
- 下部に「もっと見る」リンク（Googleカレンダー本体へ）

---

## アーキテクチャ（ざっくり）

- **Frontend**: 静的Web（1ページ）
- **Backend**: 小さなAPI（推奨）
  - 公開カレンダー（iCal/ICS）を取得して JSON に整形
  - キャッシュして配信（アクセス増にも耐える）

> ブラウザから直接ICSを取りに行くと、CORS等で詰まることがあるので  
> v0は「サーバ側で取得→JSON化→配信」に寄せる想定です。

---

## ロードマップ（予定）

- v0: イベントスケジュール（まず出す）
- v1: 写真/ライブカメラのカルーセル
- v2: 気温（観光地の体感が増す）
- v3: CMカルーセル（差し込みルール含む）

※順番は運用しながら調整します。
